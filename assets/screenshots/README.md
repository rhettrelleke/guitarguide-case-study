# Screenshots

Use this folder for curated public screenshots only.

Recommended contents:
- 2 to 4 clean product screenshots
- one optional hero image for the top-level `README.md`
- short captions that explain the point of each screen

Guidelines:
- show happy-path UI only
- avoid debug text, edge states, and hidden flows
- crop aggressively
- blur incidental labels if needed
- avoid screens that reveal internal terminology or experimental features

If a screenshot explains too much about how the product works internally, do not use it.
