# Repository Scope

This repository is a public-facing showcase for GuitarGuide.

Its job is to communicate product thinking, UX standards, and engineering judgment in a form that is clear to employers, recruiters, and curious users. It is intentionally curated. It is not a source release.

## Why This Exists
GuitarGuide is a real product with real users and a distinct point of view. A public repository should reflect that level of seriousness, but it should do so without exposing the details that make the product easy to recreate.

The goal is to show:
- how the product is framed
- what design principles shape it
- which tradeoffs matter
- what kind of engineering judgment sits behind the experience

The goal is not to show:
- how the app is organized internally
- how proprietary logic works
- how monetization is implemented
- how analytics are structured
- how to rebuild the product from public materials

## Repository Boundaries
Included here:
- high-level product explanation
- principles and decisions
- carefully abstracted diagrams
- UX notes and product rationale
- safe public-facing assets

Intentionally excluded:
- private application code
- exact architecture
- internal file names and service names
- implementation sequences
- production analytics details
- business intelligence and conversion playbooks
- internal prompts, tools, or operating docs
