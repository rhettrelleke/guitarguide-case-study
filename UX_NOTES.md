# UX Notes

## The UX Standard
The interface should help a player move from uncertainty to a usable musical decision with as little friction as possible.

This means the product should feel:
- focused
- calm
- fast enough to trust
- opinionated without feeling rigid

## What The UX Is Optimizing For

### 1. Fast Orientation
Users should understand what to do next without studying the interface. The product should make the core workflow legible quickly.

### 2. Creative Continuity
A promising idea should not disappear because the user had to navigate through too many layers or break context to evaluate it.

### 3. Lightweight Comparison
Writing often depends on comparing one musical direction against another. The experience should support that comparison without making the user feel like they are managing a system.

### 4. Feedback That Helps Decisions
Playback is valuable because it gives the user a faster way to judge whether something works. The interaction is not there to entertain. It is there to reduce uncertainty.

### 5. Depth Without Clutter
The product needs enough flexibility to stay useful, but not so much surface area that the interface starts feeling busy or abstract.

## UX Patterns Worth Protecting
- Keep the next useful action close to the current context.
- Avoid language that sounds technical when a musical phrasing would be clearer.
- Let visual hierarchy do most of the guidance work.
- Make customization support the workflow rather than distract from it.
- Treat tuning context as something that changes the experience meaningfully.

## What To Avoid
- screens that feel like reference databases
- dense controls that interrupt flow
- features that look impressive but do not support real writing
- explanations that require users to think like system operators
